HTML for Email Development

Why Tables in Email?

Unlike modern web development, email HTML relies heavily on tables for layout due to inconsistent CSS support across email clients.

Supported Elements

  • Tables & nested tables
  • Inline styles
  • Basic HTML tags
  • Limited CSS properties

Avoid These

  • External stylesheets
  • Modern CSS layouts
  • JavaScript
  • Complex CSS selectors

Basic Email Structure

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>Email Title</title>
</head>
<body>
  <table width="100%" border="0" cellspacing="0" cellpadding="0">
    <tr>
      <td align="center">
        <table width="600" border="0" cellspacing="0" cellpadding="0">
          <tr>
            <td>Content goes here</td>
          </tr>
        </table>
      </td>
    </tr>
  </table>
</body>
</html>

Standard table-based structure for email compatibility.

Email Head Requirements

<meta http-equiv="Content-Type" content="text/html; charset=utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Your Email Subject</title>

  <xml>
    <o:OfficeDocumentSettings>

    </o:OfficeDocumentSettings>
  </xml>
<![endif]-->

Essential meta tags and conditional comments for Outlook.

CSS in Email Development

Inline CSS is King

Most email clients strip or ignore CSS in the <head> section, making inline styles the most reliable method for styling emails.

Safe CSS Properties

  • color, background-color
  • font-family, font-size
  • text-align, line-height
  • padding, margin (limited)

Problematic Properties

  • position: absolute/fixed
  • float, clear
  • z-index
  • CSS Grid, Flexbox

Inline CSS Example

<table width="100%" border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td style="font-family: Arial, sans-serif; 
               font-size: 16px; 
               color: #333333;
               padding: 20px;">
      This is styled text with inline CSS
    </td>
  </tr>
</table>

Inline styles ensure consistent rendering across email clients.

Conditional CSS for Outlook

<!--[if mso]>
  <style type="text/css">
    .outlook-fix {
      mso-line-height-rule: exactly;
    }
  </style>
<![endif]-->

<table>
  <tr>
    <td class="outlook-fix">
      Content with Outlook-specific fixes
    </td>
  </tr>
</table>

Conditional comments target Microsoft Outlook specifically.

Responsive Email Design

Mobile-First Approach

With over 50% of emails opened on mobile devices, responsive design is crucial for email engagement and readability.

Responsive Techniques

  • Fluid tables with percentages
  • Media queries for mobile
  • Stacking columns on small screens
  • Touch-friendly button sizes

Mobile Considerations

  • Minimum font size: 14px
  • Button size: 44px minimum
  • Single column layout
  • Optimized image sizes

Media Query Example

<style>
  @media only screen and (max-width: 600px) {
    .container {
      width: 100% !important;
    }
    .mobile-stack {
      display: block !important;
      width: 100% !important;
    }
    .mobile-center {
      text-align: center !important;
    }
  }
</style>

Media queries adapt layout for smaller screens.

Hybrid Approach

<table class="container" width="100%">
  <tr>
    <td class="mobile-stack" width="50%">
      Column 1
    </td>
    <td class="mobile-stack" width="50%">
      Column 2
    </td>
  </tr>
</table>

<!-- On mobile, columns stack vertically -->

Hybrid coding works without media queries for maximum compatibility.

Email Testing & Debugging

Cross-Client Testing

Testing emails across different clients and devices is essential to ensure consistent rendering and user experience.

Testing Tools

  • Litmus
  • Email on Acid
  • Mailchimp Inbox Preview

Manual Testing

  • Gmail (Web & Mobile)
  • Outlook (Desktop & Web)
  • Apple Mail

Common Issues

  • Outlook image scaling
  • Gmail clipping
  • Dark mode adaptation

Pre-Send Checklist

All images have alt text

For accessibility and when images are blocked

Links are properly formatted

All URLs should be absolute with https://

Tested in multiple clients

Gmail, Outlook, Apple Mail at minimum

Mobile responsive

Layout adapts to smaller screens

Common Fixes

Outlook Image Issues

Add explicit width/height and use VML for background images

Gmail Padding Problems

Use margin instead of padding or empty table cells for spacing

Dark Mode

Use transparent PNGs and test color contrast in dark mode

Apple Mail Specific

Avoid -webkit specific properties for better compatibility
